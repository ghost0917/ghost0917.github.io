2021年EXP信息(7)

发表于2022-06-02|更新于2022-06-02

CVE-2021-0316

在 avrc_pars_tg.cc 的 avrc_pars_vendor_cmd 中，由于缺少边界检查，可能存在越界写入。这可能会导致通过蓝牙远程执行代码，而无需额外的执行权限。开发不需要用户交互。产品：安卓；版本：Android-11、Android-8.0、Android-8.1、Android-9、Android-10；安卓 ID：A-168802990。

  • https://github.com/Satheesh575555/system_bt_AOSP_10_r33_CVE-2021-0316

CVE-2021-0318

在 SensorEventConnection.cpp 的 appendEventsToCacheLocked 中，由于 use-after-free 可能存在越界写入。这可能会导致本地权限升级，而无需额外的执行权限。开发不需要用户交互。产品：安卓；版本：Android-9、Android-8.1、Android-10、Android-11；安卓 ID：A-168211968。

  • https://github.com/nanopathi/frameworks_native_AOSP10_r33_CVE-2021-0318

CVE-2021-0319

在 CompanionDeviceManagerService.java 的 checkCallerIsSystemOr 中，由于权限绕过，有可能在没有适当权限的情况下获取附近蓝牙设备的 MAC 地址。这可能导致本地权限升级，授予对附近 MAC 地址的访问权限，需要用户执行权限。开发需要用户交互。产品：安卓；版本：Android-8.0、Android-8.1、Android-9、Android-10、Android-11；安卓 ID：A-167244818。

  • https://github.com/Satheesh575555/frameworks_base_AOSP10_r33_CVE-2021-0319

CVE-2021-3118

** 分配时不支持 ** EVOLUCARE ECSIMAGING（又名 ECS Imaging）到 6.21.5 在登录表单和密码忘记表单（例如 /req_password_user.php?email=）中有多个 SQL 注入问题。这允许攻击者窃取数据库中的数据并获得对应用程序的访问权限。 （数据库组件以 root 身份运行。） 注意：此漏洞仅影响维护者不再支持的产品。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-3118

CVE-2021-3116

before_upstream_connection in AuthPlugin in http/proxy/auth.py in proxy.py before 2.3.1 接受不正确的 Proxy-Authorization 标头数据，因为布尔混淆（和与或）。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-3116

CVE-2021-21110

在 87.0.4280.141 之前的 Google Chrome 安全浏览中免费使用后，远程攻击者可以通过精心设计的 HTML 页面执行沙盒逃逸。

  • https://github.com/Gh0st0ne/CVE-2021-21110

CVE-2021-3111

Concrete5 8.5.4 中的 Express Entries Dashboard 允许通过 index.php/dashboard/express/entries/view/ URI 处的新数据对象的名称字段存储 XSS。

  • https://www.exploit-db.com/exploits/49721

  • https://nvd.nist.gov/vuln/detail/CVE-2021-3111

CVE-2021-3025

Invision Community IPS Community Suite 4.5.4.2 之前的版本允许通过下载 REST API 进行 SQL 注入（应用程序/downloads/api/files.php 中的 GETindex() 方法的 sortBy=popular 操作中的 sortDir 参数）。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-3025

CVE-2021-1056

适用于 Linux 的 NVIDIA GPU 显示驱动程序（所有版本）在内核模式层 (nvidia.ko) 中包含一个漏洞，其中它不完全尊重操作系统文件系统权限以提供 GPU 设备级隔离，这可能导致拒绝服务或信息披露。

  • https://github.com/pokerfaceSad/CVE-2021-1056

CVE-2021-23241

如 /loginLess/../../etc/passwd 所示，MERCUSYS Mercury X18G 1.0.5 设备允许通过 ../ 结合 loginLess 或 login.htm URI（用于身份验证绕过）对 Web 服务器进行目录遍历URI。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-23241

CVE-2021-23242

MERCUSYS Mercury X18G 1.0.5 设备允许通过 ../ 到 UPnP 服务器的目录遍历，如 /../../conf/template/uhttpd.json URI 所示。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-23242

CVE-2021-3011

在 NXP SmartMX / P5x 安全微控制器和 A7x 安全认证微控制器上发现了电磁波边信道问题，CryptoLib 到 v2.9。它允许攻击者在广泛的物理访问后提取 ECDSA 私钥（并因此产生克隆）。这在基于 NXP A7005a 芯片的 Google Titan 安全密钥上得到了演示。其他FIDO U2F安全密钥也受到影响（Yubico Yubikey Neo和Feian K9，K13，K21和K40）以及几种NXP JavaCard智能卡（J3A081，J2A081，J3A041，J3D145_M59，J2D145_M59，J3D120_M60，J3D082_M60，J2D120_M60，J2D082_M60，J3D081_M59 ，J2D081_M59，J3D081_M61，J2D081_M61，J3D081_M59_DF，J3D081_M61_DF，J3E081_M64，J3E081_M66，J2E081_M64，J3E041_M66，J3E016_M66，J3E016_M64，J3E041_M64，J3E145_M64，J3E120_M65，J3E082_M65，J2E145_M64，J2E120_M65，J2E082_M65，J3E081_M64_DF，J3E081_M66_DF，J3E041_M66_DF，J3E016_M66_DF，J3E041_M64_DF和J3E016_M64_DF） .

  • https://nvd.nist.gov/vuln/detail/CVE-2021-3011

CVE-2021-21236

CairoSVG 是一个 Python (pypi) 包。 CairoSVG 是一个基于 Cairo 的 SVG 转换器。在 CairoSVG 2.5.1 之前的版本中，存在正则表达式拒绝服务 (REDoS) 漏洞。在处理 SVG 文件时，python 包 CairoSVG 使用了两个容易受到正则表达式拒绝服务 (REDoS) 攻击的正则表达式。如果攻击者提供恶意 SVG，它可以使 cairosvg 在处理文件时卡住很长时间。这在 2.5.1 版中已修复。有关更多信息，请参阅参考的 GitHub 公告。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-21236

CVE-2021-21234

spring-boot-actuator-logview 库中添加了一个简单的日志文件查看器作为 spring boot 执行器端点。它是 maven 包“eu.hinsch:spring-boot-actuator-logview”。 spring-boot-actuator-logview 0.2.13 之前版本存在目录遍历漏洞。这个库的本质是通过 admin（spring boot actuator）HTTP 端点公开一个日志文件目录。可以通过请求参数指定要查看的文件名和基本文件夹（相对于日志文件夹根目录）。虽然检查了 filename 参数以防止目录遍历漏洞（因此 filename=../somefile 不起作用），但未充分检查基本文件夹参数，因此 filename=somefile&base=../ 可以访问记录基本目录之外的文件）。该漏洞已在 0.2.13 版本中修复。 0.2.12 的任何用户都应该能够毫无问题地进行更新，因为该版本中没有其他更改。除了更新或删除依赖项之外，没有解决该漏洞的解决方法。但是，删除运行应用程序的用户对运行应用程序不需要的任何目录的读取访问权限可以限制影响。此外，可以通过在反向代理后面部署应用程序来限制对 logview 端点的访问。

  • https://github.com/PwCNO-CTO/CVE-2021-21234

CVE-2021-3018

ipeak Infosystems ibexwebCMS（又名 IPeakCMS）3.5 容易通过 /cms/print.php 页面上的 id 参数受到未经身份验证的基于布尔值的 SQL 注入。

  • https://www.exploit-db.com/exploits/49372

  • https://nvd.nist.gov/vuln/detail/CVE-2021-3018

CVE-2021-3019

ffay lanproxy 0.1 允许目录遍历读取 /../conf/config.properties 以获取连接到 Intranet 的凭据。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-3019

CVE-2021-3014

在 MikroTik RouterOS 到 2021-01-04 之前，热点登录页面容易受到通过 target 参数反射的 XSS 的攻击。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-3014

CVE-2021-21494

MK-AUTH 到 19.01 K4.9 允许通过 admin/logs_ajax.php 的 Tipo 参数进行 XSS。攻击者可以利用它来读取未设置为 HTTPOnly 的 centralmka2（会话令牌）cookie。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-21494

CVE-2021-21495

MK-AUTH 到 19.01 K4.9 允许 CSRF 通过 central/executar_central.php?acao=altsenha_princ URI 更改密码。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-21495

CVE-2021-3007

** DISPUTED ** Laminas 项目 laminas-http 2.14.2 之前和 Zend Framework 3.0.0 存在反序列化漏洞，如果内容可控可导致远程代码执行，与 Zend\Http\ 的 __destruct 方法有关Stream.php 中的 Response\Stream 类。注意：Zend Framework 不再被维护者支持。注意：laminas-http 供应商认为这是“PHP 语言本身的漏洞”，但添加了某些类型检查，以防止在攻击者提供的数据可以反序列化的（不推荐的）用例中被利用。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-3007

CVE-2021-3006

以太坊代币 Farm in Seal Finance (Seal) 的智能合约实施中的品种功能缺乏访问控制，因此允许价格操纵，正如 2020 年 12 月和 2021 年 1 月在野外利用的那样。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-3006

CVE-2021-3004

以太坊代币稳定收益信用（yCREDIT）的智能合约实现中的_deposit函数存在某些错误计算。攻击者可以获得比他们应得的更多的 yCREDIT 令牌。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-3004

CVE-2021-3002

Seo Panel 4.8.0 允许通过 seo/seopanel/login.php?sec=forgot email 参数反射 XSS。

  • https://nvd.nist.gov/vuln/detail/CVE-2021-3002
