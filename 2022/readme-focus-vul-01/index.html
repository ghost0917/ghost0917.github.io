CVE-2022-26670

CVE-2022-20665

Cisco StarOS 的 CLI 中的漏洞可能允许经过身份验证的本地攻击者提升受影响设备的权限。此漏洞是由于 CLI 命令的输入验证不足所致。攻击者可以通过向 CLI 发送精心设计的命令来利用此漏洞。成功的利用可能允许攻击者以 root 用户的权限执行任意代码。要利用此漏洞，攻击者需要在受影响的设备上拥有有效的管理凭据。

CVE-2022-20774

CVE-2022-20650

Cisco NX-OS 软件的 NX-API 功能中的一个漏洞可能允许经过身份验证的远程攻击者以 root 权限执行任意命令。该漏洞是由于对发送到 NX-API 的用户提供的数据的输入验证不足所致。攻击者可以通过向受影响设备的 NX-API 发送精心设计的 HTTP POST 请求来利用此漏洞。成功的利用可能允许攻击者在底层操作系统上以 root 权限执行任意命令。注意：默认情况下禁用 NX-API 功能。

CVE-2022-20625

思科 FXOS 软件和思科 NX-OS 软件的思科发现协议服务中的漏洞可能允许未经身份验证的相邻攻击者导致服务重新启动，从而导致拒绝服务 (DoS) 状况。此漏洞是由于对思科发现协议服务处理的思科发现协议消息处理不当造成的。攻击者可以通过向受影响的设备发送一系列恶意思科发现协议消息来利用此漏洞。成功的利用可能允许攻击者导致 Cisco 发现协议服务失败并重新启动。在极少数情况下，可能会发生该过程的重复失败，这可能导致整个设备重新启动。

CVE-2022-20624

Cisco NX-OS 软件的 Cisco Fabric Services over IP (CFSoIP) 功能中的一个漏洞可能允许未经身份验证的远程攻击者在受影响的设备上造成拒绝服务 (DoS) 状况。此漏洞是由于对传入 CFSoIP 数据包的验证不足所致。攻击者可以通过向受影响的设备发送精心设计的 CFSoIP 数据包来利用此漏洞。成功的利用可能允许攻击者导致受影响的设备重新加载，从而导致 DoS 条件。

CVE-2022-20623

用于 Cisco Nexus 9000 系列交换机的 Cisco NX-OS 软件的双向转发检测 (BFD) 流量的速率限制器中的漏洞可能允许未经身份验证的远程攻击者导致 BFD 流量在受影响的设备上被丢弃。此漏洞是由于 BFD 速率限制器功能中的逻辑错误造成的。攻击者可以通过设备发送精心设计的流量流来利用此漏洞。成功的利用可能允许攻击者导致 BFD 流量被丢弃，从而导致 BFD 会话抖动。 BFD 会话波动会导致路由不稳定和流量下降，从而导致拒绝服务 (DoS) 情况。此漏洞适用于 IPv4 和 IPv6 流量。

CVE-2022-20658

Cisco Unified Contact Center Management Portal (Unified CCMP) 和 Cisco Unified Contact Center Domain Manager (Unified CCDM) 基于 Web 的管理界面中存在漏洞，可能允许经过身份验证的远程攻击者将其权限提升为管理员。此漏洞是由于缺乏对用户权限的服务器端验证。攻击者可以通过向易受攻击的系统提交精心设计的 HTTP 请求来利用此漏洞。成功的利用可能允许攻击者创建管理员帐户。使用这些帐户，攻击者可以访问和修改与易受攻击的 Cisco Unified CCMP 关联的所有统一平台上的电话和用户资源。要成功利用此漏洞，攻击者需要有效的高级用户凭据。

CVE-2022-20660

多个 Cisco IP 电话型号的信息存储架构中的漏洞可能允许未经身份验证的物理攻击者从受影响的设备获取机密信息。此漏洞是由于受影响设备上的机密信息未加密存储造成的。攻击者可以通过物理提取和访问其中一个闪存芯片来利用此漏洞。成功的利用可能允许攻击者从设备获取机密信息，这些信息可用于后续攻击。
