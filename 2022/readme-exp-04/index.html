CVE-2022-21668

pipenv 是一个 Python 开发工作流工具。从 2018.10.9 版本开始到 2022.1.8 版本之前，pipenv 解析需求文件中的一个缺陷允许攻击者在 requirements.txt 文件中的任何位置的注释中插入特制字符串，这将导致使用 pipenv 的受害者安装需求文件以从攻击者控制的包索引服务器下载依赖项。通过在恶意索引服务器提供的包中嵌入恶意代码，攻击者可以在受害者的系统上触发任意远程代码执行 (RCE)。如果攻击者能够在受害者使用 pipenv 安装的需求文件中隐藏恶意 --index-url 选项，则攻击者可以将任意恶意代码嵌入从其恶意索引服务器提供的包中，这些代码将在受害者的安装期间的主机（远程代码执行/RCE）。当 pip 从源分发安装时， setup.py 中的任何代码都由安装过程执行。此问题已在版本 2022.1.8 中修复。 GitHub 安全公告包含有关此漏洞的更多信息。

  • https://github.com/sreeram281997/CVE-2022-21668-Pipenv-RCE-vulnerability

  • https://nvd.nist.gov/vuln/detail/CVE-2022-21668

CVE-2022-21670

markdown-it 是一个 Markdown 解析器。在 1.3.2 版本之前，长度超过 50,000 个字符的特殊模式可能会显着降低解析器的速度。用户应升级到版本 12.3.2 以接收补丁。除了升级之外，没有已知的解决方法。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-21670

CVE-2022-22114

在 Teedy 中，v1.5 到 v1.9 版本容易受到反射跨站点脚本 (XSS) 的攻击。 “搜索词”搜索功能在显示搜索结果时没有得到充分的净化，可被用来注入任意脚本。当受害者输入精心设计的 URL 时，这些脚本会在受害者的浏览器中执行。在最坏的情况下，受害者无意触发攻击的是一个高权限的管理员，注入的脚本可以提取会话ID，这可能导致管理员的帐户完全接管，由未经身份验证的攻击者。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-22114

CVE-2022-22115

在 Teedy 中，v1.5 到 v1.9 版本容易受到以创建的标签为名的存储跨站点脚本 (XSS) 的攻击。由于标签名称未在编辑标签页面中正确清理，低权限攻击者可以将恶意脚本存储在标签名称中。在最坏的情况下，无意中触发攻击的受害者是具有高度特权的管理员。注入的脚本可以提取会话 ID，这可能导致管理员完全接管帐户并提升权限。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-22115

CVE-2022-22116

在 Directus 中，版本 9.0.0-alpha.4 到 9.4.1 容易受到通过媒体上传功能中的 SVG 文件上传存储的跨站点脚本 (XSS) 漏洞的攻击。低权限的攻击者可以注入任意 javascript 代码，当受害者打开图像 URL 时，这些代码将在受害者的浏览器中执行。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-22116

CVE-2022-22117

在 Directus 中，版本 9.0.0-alpha.4 到 9.4.1 允许在媒体上传功能中不受限制地上传 .html 文件，这会导致跨站点脚本漏洞。低权限攻击者可以上传精心制作的 HTML 文件作为配置文件头像，当管理员或其他用户打开它时，XSS 有效负载会被触发。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-22117

CVE-2022-22120

在 NocoDB 中，0.9 到 0.83.8 版本容易受到密码重置功能中的可观察差异的影响。当请求为给定电子邮件地址重置密码时，如果电子邮件未在系统中注册，应用程序会显示错误消息。这允许攻击者枚举注册用户的电子邮件地址。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-22120

CVE-2022-22121

在 NocoDB 中，0.81.0 到 0.83.8 版本受到 CSV 注入漏洞（公式注入）的影响。低权限攻击者可以创建一个新表以在表行中注入有效负载。当管理员访问用户管理端点并将数据导出为 CSV 文件并打开它时，有效负载将被执行。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-22121

CVE-2022-22844

在某些情况下，LibTIFF 4.3.0 在 tif_unix.c 中的 _TIFFmemcpy 中读取越界，涉及自定义标签和 0x0200 作为 DE 字段的第二个字。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-22844

CVE-2022-22701

  • https://nvd.nist.gov/vuln/detail/CVE-2022-22701

CVE-2022-22702

直到 v1.4.0 的 PartKeepr 版本，在创建部件时使用 URL 上传附件的功能不会验证是否可以向本地端口发出请求，从而允许经过身份验证的用户执行 SSRF 攻击和端口枚举。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-22702

CVE-2022-21661

WordPress 是一个免费的开源内容管理系统，用 PHP 编写并与 MariaDB 数据库配对。由于 WP_Query 中的不当清理，可能存在通过以某种方式使用它的插件或主题进行 SQL 注入的情况。这已在 WordPress 版本 5.8.3 中进行了修补。受影响的旧版本也通过安全版本修复，可以追溯到 3.7.37。我们强烈建议您保持启用自动更新。此漏洞没有已知的解决方法。

  • https://www.exploit-db.com/exploits/50663

  • https://github.com/0x4E0x650x6F/Wordpress-cve-CVE-2022-21661

  • https://github.com/PyterSmithDarkGhost/CVE-2022-21661-WordPress-Core-5.8.2-WP_Query-SQL-Injection-main

  • https://nvd.nist.gov/vuln/detail/CVE-2022-21661

CVE-2022-22707

在 lighttpd 1.4.46 到 1.4.63 中，mod_extforward 插件的 mod_extforward_Forwarded 函数具有基于堆栈的缓冲区溢出（4 个字节表示 -1），如非默认配置中的远程拒绝服务（守护进程崩溃）所示。非默认配置需要以某种不寻常的方式处理 Forwarded 标头。此外，32 位系统比 64 位系统更容易受到影响。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-22707

CVE-2022-21653

Jawn 是一个开源的 JSON 解析器。不覆盖 objectContext() 的 org.typelevel.jawn.SimpleFacade 和 org.typelevel.jawn.MutableFacade 的扩展程序容易受到哈希冲突攻击，这可能导致拒绝服务。大多数应用程序不直接实现这些特征，而是从库继承。 jawn-parser-1.3.1 修复了这个问题，建议用户升级。对于无法升级覆盖 objectContext() 以使用碰撞安全集合的用户。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-21653

CVE-2022-21649

Convos 是一个在网络浏览器中运行的开源多用户聊天。聊天窗口中以“https://”开头的字符会创建一个 <a> 标签。使用 onfocus 和 autofocus 的存储型 XSS 漏洞发生是因为“<”或“>”存在转义但不存在双引号的转义。通过此漏洞，攻击者能够执行恶意脚本。建议用户尽快更新。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-21649

CVE-2022-21650

Convos 是一个在网络浏览器中运行的开源多用户聊天。您不能在 Convos 的聊天窗口中使用 SVG 扩展名，但您可以上传带有 .html 扩展名的文件。通过上传带有 html 扩展名的 SVG 文件，可以绕过上传过滤器。这会导致存储型 XSS。此外，在上传文件后，XSS 攻击会在用户查看文件时触发。通过此漏洞，攻击者能够执行恶意脚本。建议用户尽快更新。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-21650
