2022年CVE&EXP信息(7)

发表于2022-05-23|更新于2022-05-23
|阅读量:

CVE-2022-22111

在 DayByDay CRM 中，版本 2.2.0 容易丢失授权。应用程序中启用更新用户权限的任何应用程序用户都可以更改其他用户的密码，包括管理员的密码。这允许攻击者访问应用程序中的最高权限用户。

CVE-2022-21649

Convos 是一个在网络浏览器中运行的开源多用户聊天。聊天窗口中以“https://”开头的字符会创建一个 <a> 标签。使用 onfocus 和 autofocus 的存储型 XSS 漏洞发生是因为“<”或“>”存在转义但不存在双引号的转义。通过此漏洞，攻击者能够执行恶意脚本。建议用户尽快更新。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-21649

CVE-2022-21650

Convos 是一个在网络浏览器中运行的开源多用户聊天。您不能在 Convos 的聊天窗口中使用 SVG 扩展名，但您可以上传带有 .html 扩展名的文件。通过上传带有 html 扩展名的 SVG 文件，可以绕过上传过滤器。这会导致存储型 XSS。此外，在上传文件后，XSS 攻击会在用户查看文件时触发。通过此漏洞，攻击者能够执行恶意脚本。建议用户尽快更新。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-21650

CVE-2022-21643

USOC 是一个专注于简单性的开源 CMS。在受影响的版本中，USOC 允许通过 register.php 进行 SQL 注入。特别是用户提供的用户名、电子邮件地址和密码没有经过清理，而是直接用于构建 sql 语句。建议用户尽快升级。此问题没有解决方法。

CVE-2022-21644

USOC 是一个专注于简单性的开源 CMS。在受影响的版本中，USOC 允许通过 usersearch.php 进行 SQL 注入。用户提供的搜索词没有经过过滤，直接用于构建 sql 语句。唯一允许搜索的用户是站点管理员。建议用户尽快升级。此问题没有解决方法。

CVE-2022-21647

CodeIgniter 是一个开源的 PHP 全栈 Web 框架。在 CodeIgniter4 的 old() 函数中发现了不可信数据的反序列化。远程攻击者可能会利用此漏洞注入可自动加载的任意对象，并可能在服务器上执行现有的 PHP 代码。我们知道一个有效的漏洞利用，它可能导致 SQL 注入。建议用户升级到 v4.1.6 或更高版本。用户无法升级，建议不要使用 old() 函数和 form_helper，也不要使用 RedirectResponse::withInput() 和 redirect()->withInput()。

CVE-2022-21648

Latte 是 PHP 的开源模板引擎。自 2.8.0 Latte 起的版本包含一个模板沙箱，在受影响的版本中，发现存在沙箱逃逸，允许注入从 Latte 生成的网页中。这可能会导致 XSS 攻击。该问题已在 2.8.8、2.9.6 和 2.10.8 版本中得到修复。无法升级的用户不应接受来自不受信任来源的模板输入。

  • https://nvd.nist.gov/vuln/detail/CVE-2022-0086
